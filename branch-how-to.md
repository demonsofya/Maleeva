# Работа с ветками в Git

## Как создать ветку

### 1. Создание ветки
# Создать ветку с именем "new-feature"
git branch new-feature

### 2. Создание и переключение
# Создать и сразу переключиться
git checkout -b new-feature
# Или в новых версиях Git:
git switch -c new-feature

## Как переключиться в ветку
# Переключиться на существующую ветку
git checkout имя_ветки
# Или:
git switch имя_ветки

# Посмотреть все ветки
git branch

# Посмотреть все ветки (включая удаленные)
git branch -a

## Как отправить ветку на сервер

### 1. Первая отправка ветки
# Отправить ветку на сервер (первый раз)
git push -u origin имя_ветки

### 2. Последующие отправки
# После установки связи с удаленной веткой
git push

### 3. Получение изменений с сервера
# Получить изменения из удаленной ветки
git pull origin имя_ветки

## Пример работы процесса

1. Создание ветки для новой функции:
  
   git checkout -b add-login-feature
   
2. Работа над изменениями:
  
   # Добавление файлов
   git add .
   git commit -m "Add login form"
   
3. Отправка на сервер:
  
   git push -u origin add-login-feature
